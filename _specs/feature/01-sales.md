# Sales / Quoting / Ordering

## §1 試算・価格・見積

### 機能概要

試算（原価計算）→ 価格表 → 見積書 の 3 段フロー。フロー起点は**試算**：原価（材料費・加工費・外注費・段取り費）と利益率から数量区分ごとの単価を算出し、確定した試算を**価格表として登録**する。**見積書は価格表から自動生成（印刷）**し、必要に応じてカスタム値引き（値引き額）を適用する。発行時に生成した PDF は保存され、見積書詳細の PDF タブで閲覧できる。

```
試算（原価計算・単価決定）─登録→ 価格表（顧客×製品×数量区分の単価）─印刷→ 見積書（PDF 保存・閲覧）
```

### 画面

| パス | 内容 |
|------|------|
| `/sales/estimates` | 試算一覧（ステータス・顧客・製品フィルタ） |
| `/sales/estimates/new` | 試算新規作成（原価計算フォーム＋数量区分別単価のライブ計算） |
| `/sales/estimates/[id]` | 試算詳細（計算内訳・価格表登録アクション） |
| `/sales/estimates/[id]/edit` | 試算編集 |
| `/sales/price-lists` | 価格表一覧（顧客・製品・有効日フィルタ） |
| `/sales/price-lists/new` | 価格表新規作成（手動登録 — 通常は試算から登録） |
| `/sales/price-lists/[id]` | 価格表詳細（試算元参照・見積書作成アクション） |
| `/sales/price-lists/[id]/edit` | 価格表編集 |
| `/sales/quotes` | 見積書一覧（ステータス・顧客フィルタ、PGroonga 全文検索） |
| `/sales/quotes/new` | 見積書新規作成 |
| `/sales/quotes/[id]` | 見積書詳細（明細・PDF・関連・履歴タブ） |
| `/sales/quotes/[id]/edit` | 見積書編集 |

### 主要機能

- **試算（原価計算）**: 顧客＋製品＋注文種別に対し、材料費/本・加工時間×加工レート・外注費/本・段取り費（ロット按分）を入力し、数量区分ごとに `原価/本` を自動計算。`単価 = 原価 ÷ (1 − 利益率)`（10 円単位切り上げ、手動調整可）。計算ロジックは `lib/pricing.ts`。
- 試算採番: `EST-YYYYMM-NNNNN`（`lib/numbering.ts`）
- 試算ステータス管理: `DRAFT → CONFIRMED → REGISTERED`（価格表登録済）
- **価格表への登録**: 確定した試算の数量区分別単価を、有効期間を指定して価格表行として一括登録（`price_lists.estimate_id` で試算元を保持）。
- 価格表: 顧客＋製品コード＋注文種別（本番/テスト/サンプル/その他）＋本数範囲 = 単価、有効日管理
- **見積書の自動生成（印刷）**: 顧客・製品・注文種別・本数から価格表を解決し、見積明細（単価・金額）を自動生成。営業は数量入力のみで明細が確定する（`lib/pricing.ts`）。価格表詳細からは「見積書を作成」で数量入力のみのワンステップ起票が可能。
- **カスタム値引き（任意）**: 明細ごとに値引き額（`discount_amount`）を入力可能。`金額 = 単価 × 本数 − 値引き額` で再計算。値引きは必要時のみ入力（未入力時は 0）。
- 見積書採番: `QOT-YYYYMM-NNNNN`（`lib/numbering.ts`）
- 見積書 PDF 生成: `app/api/pdf/quote/route.ts` → Gotenberg。**発行時に生成した PDF は SeaweedFS に保存し `quotes.pdf_file_id` で参照**。見積書詳細の **PDF タブ**でインライン閲覧・ダウンロード・再生成できる。
- 見積書ステータス管理: `DRAFT → ISSUED → ACCEPTED / REJECTED / EXPIRED`
- 設計依頼書への分岐: 設計図なしの場合 §10 へ起票リンク

### 業務ルール

- 注文種別: 本番 / テスト / サンプル（金額 0）/ その他
- 試算は `CONFIRMED`（確定）にしてから価格表登録が可能。登録後は `REGISTERED` となり原価項目は編集不可（再試算は複製で行う）
- 段取り費は数量区分の下限本数で按分する（`段取り費 ÷ min_quantity`）
- 価格表は有効日範囲で管理（重複・空白は検証で防止）。試算から登録した行は `estimate_id` で試算元を参照（手動登録時は null）
- 見積明細は価格表から自動生成する。価格表に該当行がない（新規品等）場合は単価未解決として警告し、試算（新規作成）または §10 設計依頼へ誘導
- 値引きはカスタム入力（任意）。値引き後金額がマイナスにならないよう検証
- 見積書 PDF は発行（`ISSUED`）時に確定保存する。再生成すると `files` に新規行を作成し `pdf_file_id` を差し替え（旧ファイルは監査用に保持）
- 見積書から注文請書への転記時のコピー対象を制御（新規品は除外）

## §2 注文受付・価格差異

### 機能概要

顧客注文書受領から受注請書作成、価格差異照合。

### 画面

| パス | 内容 |
|------|------|
| `/sales/order-acceptances` | 受注請書一覧（ステータス・顧客フィルタ） |
| `/sales/order-acceptances/intake` | 受注請書 取込〜受諾（AI 読取 → 確認・修正 → 受諾） |
| `/sales/order-acceptances/new` | 受注請書新規作成 |
| `/sales/order-acceptances/[id]` | 受注請書詳細（注文書 PDF 表示含む） |
| `/sales/order-acceptances/[id]/edit` | 受注請書編集 |

### 主要機能

- 受注請書採番: `ORD-YYYYMM-NNNNN`
- 注文書 PDF アップロード・添付（SeaweedFS）
- AI 取込: 注文書 PDF / スキャンを `POST /extract/order-request`（ai-stack po-extract）で構造化 JSON 化し、編集可能フォームへ自動入力（明細は品番・版数・カスタム内容・届け先まで抽出）
- 受諾フロー: 読取結果の確認・修正 → 受諾ステップ（確定内容の読み取り専用プレビュー + 作成される注文請書・指示書の内訳表示）→ 受諾確定
- 合計金額の自動計算: 注文請書の製品・本数・単価から自動集計（手動編集不可）
- 価格差異自動照合: 注文価格と見積価格の比較、差異時は担当者へ通知（SSE / メール）
- ステータス管理: `PENDING → PRICE_DIFF / CONFIRMED`

### 業務ルール

- 顧客は企業・支店の 2 階層
- 合計金額は注文請書から自動計算
- 価格差異あり時: 営業担当者に通知し見積を再調整後 §3 へ
- 受諾（確定）時の伝票展開: 受注請書 1 件 → 注文請書 1 件 + 指示書 N 件
  - 指示書は明細行を「製品・版数 / カスタム内容 / 納期 / 届け先」の組み合わせでグルーピングし 1 件ずつ作成（同一グループの数量は合算）
  - 明細行の届け先が未指定の場合はヘッダの受渡場所、納期が未指定の場合は希望納期を採用

## §3 注文請書・指示書

### 機能概要

注文請書と指示書（製造ワークフロー）の作成。

### 画面

| パス | 内容 |
|------|------|
| `/production/sales-orders` | 注文請書一覧 |
| `/production/sales-orders/new` | 注文請書新規作成 |
| `/production/sales-orders/[id]` | 注文請書詳細（注文書 PDF サイドパネル表示） |
| `/production/sales-orders/[id]/edit` | 注文請書編集 |
| `/production/work-orders` | 指示書一覧（ステータス・日付フィルタ） |
| `/production/work-orders/new` | 指示書新規作成（工程ワークフロービルダー） |
| `/production/work-orders/[id]` | 指示書詳細（工程ワークフロー表示） |
| `/production/work-orders/[id]/edit` | 指示書編集 |

### 主要機能

- 注文請書採番: `ORD-YYYYMM-NNNNN-NN`（受注請書コード + 連番）
- 注文請書 PDF 生成: `app/api/pdf/sales-order/route.ts`
- ロット番号割当: 通し連番（指示書番号と共用）
- 指示書ワークフロービルダー: 工程マスタから工程を選択し順序・同期設定
- 指示書コピー機能: 同一受注元・製品の最新指示書からのコピー（バージョン変更時は警告）
- 計画本数入力: ワークフロー作成時に入力
- 外注・社内の実施場所設定: 工程ごとに社内 / 外注（企業選択）を指定
- 指示書タイプ: `FROM_STOCK`（在庫分）/ `MANUFACTURE`（製造分）
- 使用依存・実行依存バリデーション: `lib/workflow.ts` で依存関係を検証
- 検査表テンプレートを指示書に複数紐付け: `work_order_inspection_templates`

### 業務ルール

- 製品ごとに注文請書 1 行
- ロット番号・本数は必須（定尺材は伝票コード入力）
- 半製品（外部調達）には指示書不要 — 素材受入のみ
