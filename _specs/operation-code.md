# 操作コード（画面番号）

キーボードから **4文字の操作コード** で任意の画面へ直接ジャンプする機能。

実装: `design-preview/designs/lib/operation-codes.ts`（本番では `src/lib/operation-codes.ts` に移設）

---

## 形式

```
{CAT}{MODE}{IDX}
```

| 部分 | 内容 | 値域 | 例 |
|------|------|------|-----|
| `CAT` | カテゴリ（英字2文字） | `CM` `SA` `PU` `PD` `SH` `BL` `MS` | `SA` |
| `MODE` | 画面モード（1文字） | `0` 一覧 / `1` 新規 / `2` 詳細 | `2` |
| `IDX`  | 画面インデックス（1文字） | `0`–`9`、超過時 `A`–`Z`（最大36件） | `A` |

**固定4文字。** 使用可能文字: 英字（A–Z）・数字（0–9）。入力は大文字に正規化。

### カテゴリ接頭辞

| 接頭辞 | 区分 |
|--------|------|
| `CM` | 共通 |
| `SA` | 販売 |
| `PU` | 購買 |
| `PD` | 生産 |
| `SH` | 出荷 |
| `BL` | 請求 |
| `MS` | マスタ |

### 画面モード（3文字目）

| 値 | モード | URL 例 |
|----|--------|--------|
| `0` | 一覧 | `/sales/quotes` |
| `1` | 新規作成 | `/sales/quotes/new` |
| `2` | 詳細 | `/sales/quotes/[id]` — IDなし時は検索画面 |

新しいモードを追加する場合は `3`–`9` を順に割り当てる（例: `3` 編集、`4` 印刷プレビュー など、必要に応じて）。

### 画面インデックス（4文字目）

カテゴリ内の連番。

| 値 | 対応 |
|----|------|
| `0` | （予約：将来のカテゴリトップ等） |
| `1`–`9` | 1〜9件目 |
| `A`–`Z` | 10〜35件目 |

**10件以下なら数字のみ**で表現可能。10件目以降は `A` から英字を使う（旧仕様の `20–29` overflow は廃止）。

例: マスタの10件目（承認グループ）
- `MS0A` 一覧
- `MS1A` 新規
- `MS2A` 詳細

### 派生コードの導出

同じ画面の別モードは **MODE 桁を差し替えるだけ**:

| 一覧 | 新規 | 詳細 |
|------|------|------|
| `SA02` | `SA12` | `SA22` |
| `MS0A` | `MS1A` | `MS2A` |

## 詳細モード（`2`）の動作

詳細コード（例 `SA22` 見積書詳細）の遷移ルール:

| 状況 | 遷移先 | 例 |
|------|--------|----|
| URL の `id` パラメータが解決可能 | 詳細画面 | `/sales/quotes/abc-123` |
| `id` が無い・無効 | **検索画面** | `/sales/quotes/_search` |

検索画面は当該リソースから1件を選んで詳細へ進むためのピッカー UI。
キーボードから `SA22 Enter` と入力したユーザーは、まず検索画面で対象の見積書を絞り込み、選択すると詳細画面へ遷移する。

**実装方針:**
- ルート: `app/(dashboard)/{path}/_search/page.tsx`
- もしくは一覧画面に `?mode=select` クエリを付与して「選択モード」を有効化（行クリックで詳細へ jump）
- 戻り先: `searchParams.from=op:SA22` を保存し、選択完了後は元の操作フローを継続できるようにする

## 改訂版コード（将来）

画面 UI を全面リニューアルした際は **末尾に `N` を付加**（5文字）:

```
PD02     現行の指示書一覧
PD02N    改訂版の指示書一覧
```

UI 入力欄は通常4文字で Enter 自動確定、`N` が続く場合のみ5文字まで受理。

## 追加ルール

### 新しい画面を追加する

1. カテゴリ内で未使用の IDX を割り当て（`1`–`9`、超過時 `A`–`Z`）
2. 一覧 `0X`・新規 `1X`・詳細 `2X` の3コードを同時登録
3. 検索画面（詳細 `2X` の ID 無し時用）を実装

### 画面改訂時

1. `{CAT}{MODE}{IDX}N` を登録（例: `PD02N`）
2. 旧コードはリダイレクトまたは一定期間併存

## API

```ts
import {
  resolveOperationCode,
  navigateByOperationCode,
  toNewOperationCode,
  toDetailOperationCode,
  toListOperationCode,
  toUpdatedOperationCode,
  getListOperationCode,
} from '@/lib/operation-codes';

resolveOperationCode('PD02');         // 指示書一覧の Entry
toNewOperationCode('PD02');           // → 'PD12'
toDetailOperationCode('PD02');        // → 'PD22'
toDetailOperationCode('PD02', 'wo-1'); // → { code: 'PD22', href: '/production/work-orders/wo-1' }
toListOperationCode('PD22');          // → 'PD02'（モード桁を 0 に戻す）
toUpdatedOperationCode('PD02');       // → 'PD02N'（将来）
getListOperationCode('/sales/quotes'); // → 'SA02'
```

本番実装時は RBAC（`user_permissions`）を照合すること。
