# 顧客 — 操作マニュアル

操作コード **MS01**。注文をくれるお客様（受注元）を登録・管理する台帳です。見積書や価格表は、ここで登録した顧客を選んで作成します。

## このアプリでできること

「どの会社と取引しているか」をまとめて管理します。会社の基本情報に加えて、締日や支払条件などの **取引条件** を登録しておけます。

- 顧客を登録すると、[価格表](/docs/apps/price-list/user) や [見積書](/docs/apps/quote/user) で選べるようになります。
- 会社の下に **支店** を登録できます（本社 → 支店の 2 階層まで）。
- 会社ごとに **担当者** を複数登録できます。

顧客は社内で「取引先（BP）」として管理され、そのうち **顧客ロール** が付いたものです。

## 一覧を見る

- 一覧には登録済みの顧客が並びます。行をクリックすると詳細画面が開きます。
- 上部の検索ボックスに **社名・コード** を入れて絞り込めます。
- **状態**（有効 / 無効）でも絞り込めます。

## 新規作成

一覧右上の「新規作成」から登録します。主な入力項目は次のとおりです。

**基本情報**

- **名称**（日本語は必須、英語は任意）。
- **国** / **フリガナ** / **略称** / **法人番号**。
- **有効** … オフにすると選択肢に出なくなります（過去データは残ります）。
- **AI照合名** … 注文書の自動読み取り（AI 抽出）が社名をこの顧客に結び付けるための照合リストです。「㈱／株式会社」「全角／半角」「旧社名」などの表記ゆれを Enter 区切りで登録します。

**住所・連絡先**

- **郵便番号** / **住所**（日本語・英語）/ **電話番号** / **FAX** / **メールアドレス** / **Webサイト** / **備考**。

**取引条件**

- **請求先** … 請求先が別の会社の場合に指定します（未指定ならこの顧客自身に請求）。
- **締日** … 1〜31 の日付。`31` は「月末」を意味します。
- **支払サイト（日数）** / **支払日** / **与信限度額**。
- **課税区分** … 課税 / 非課税 / 軽減税率。
- **請求書送付方法** … メール / FAX / 郵送 / ポータル。
- **委託先** … 委託販売の対象ならチェックします。

**BPコード**（`BP-NNNNN`）は保存時に自動で採番されます。手入力は不要です。

## 詳細画面

詳細画面はタブに分かれています。

- **概要** … 取引条件と担当者の一覧。担当者はここで追加・編集できます。
- **支店一覧** … この顧客に属する支店の一覧。「支店を追加」から登録します。
- **見積・受注履歴** … この顧客に関係する見積・受注の履歴。
- **履歴** … 変更の記録（いつ・誰が更新したか）。

右上のメニューから **編集** / **無効化** / **削除** ができます。

## 支店の登録

- 詳細画面の「支店一覧」タブ →「支店を追加」で登録します。
- 支店も顧客と同じ基本情報を入力します。新規登録時は **主担当者名** も一緒に登録できます（任意）。
- 支店コードは `親コード-NN`（例 `BP-00001-01`）で自動採番されます。

## 用語

- **BP（取引先）** … 顧客・仕入先・需要家などをまとめて管理する会社の単位。顧客は「顧客ロール」が付いた BP です。
- **支店** … 本社の下にぶら下がる拠点。2 階層まで登録できます。
- **締日 / 支払サイト** … 請求をまとめる日と、支払までの猶予日数。
- **委託先** … 委託販売の対象となる取引先。
- **AI照合名** … 注文書の自動読み取りで社名を照合するための別名リスト。

初めての方は [スタートマニュアル](/docs/start) もあわせてご覧ください。
