# 客户 — 操作手册

操作码 **MS01**。用于登记和管理向您下单的客户（受订方）的台账。报价单与价格表都通过选择此处登记的客户来创建。

## 本应用能做什么

集中管理“与哪些公司有业务往来”。除公司基本信息外，还可登记结算日、付款条件等 **交易条件**。

- 登记客户后，即可在 [价格表](/docs/apps/price-list/user) 和 [报价单](/docs/apps/quote/user) 中选择。
- 可在公司下登记 **分店**（本部 → 分店，最多 2 层）。
- 每家公司可登记多个 **联系人**。

客户在系统内作为“业务伙伴（BP）”管理，其中带有 **客户角色** 的即为客户。

## 查看列表

- 列表显示已登记的客户。点击行可打开详情页。
- 使用顶部搜索框，按 **公司名称・代码** 筛选。
- 也可按 **状态**（有效 / 无效）筛选。

## 新建

从列表右上的“新建”登记。主要输入项如下。

**基本信息**

- **名称**（日文必填，英文选填）。
- **国家** / **假名读音** / **简称** / **法人编号**。
- **有效** … 关闭后不再出现在选择列表中（历史数据保留）。
- **AI 匹配名** … 供订单文档自动读取（AI 抽取）将公司名解析到该客户的匹配列表。将表述差异（㈱／株式会社、全角／半角、旧公司名等）以 Enter 分隔登记。

**地址・联系方式**

- **邮编** / **地址**（日文・英文）/ **电话** / **FAX** / **邮箱** / **网站** / **备注**。

**交易条件**

- **收票方** … 当由其他公司收票时指定（未指定则向该客户本身收票）。
- **结算日** … 1〜31 的日期。`31` 表示“月末”。
- **付款账期（天数）** / **付款日** / **授信额度**。
- **课税区分** … 课税 / 非课税 / 减免税率。
- **发票送付方式** … 邮件 / FAX / 邮寄 / 门户。
- **委托方** … 若为委托销售对象则勾选。

**BP 代码**（`BP-NNNNN`）在保存时自动编号，无需手动输入。

## 详情页

详情页分为多个标签。

- **概要** … 交易条件与联系人列表。联系人可在此添加、编辑。
- **分店列表** … 该客户下的分店列表。通过“添加分店”登记。
- **报价・受订历史** … 与该客户相关的报价与受订历史。
- **历史** … 变更记录（何时、由谁更新）。

通过右上菜单可进行 **编辑** / **停用** / **删除**。

## 分店登记

- 在详情页“分店列表”标签 →“添加分店”登记。
- 分店与客户输入相同的基本信息。新建时还可同时登记 **主联系人姓名**（选填）。
- 分店代码按 `父代码-NN`（例 `BP-00001-01`）自动编号。

## 术语

- **BP（业务伙伴）** … 将客户、供应商、需求方等统一管理的公司单位。客户即带有“客户角色”的 BP。
- **分店** … 隶属本部的据点，最多可登记 2 层。
- **结算日 / 付款账期** … 汇总账单的日期，以及到付款的宽限天数。
- **委托方** … 作为委托销售对象的伙伴。
- **AI 匹配名** … 订单文档自动读取时用于匹配公司名的别名列表。
