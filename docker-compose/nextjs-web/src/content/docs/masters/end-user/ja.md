# 最終需要家 — 操作マニュアル

操作コード **MS02**。製品を実際に使う会社（最終ユーザー）を登録・管理する台帳です。

## このアプリでできること

注文をくれる[顧客](/docs/masters/customer/user)（受注元）とは別に、その製品を **実際に使う会社** を記録しておきたい場合に使います。

- 登録は **任意** です。主に大口の取引で、需要家をきちんと管理したいときに使います。
- 最終需要家は社内で「取引先（BP）」として管理され、そのうち **需要家ロール** が付いたものです。

## 一覧を見る

- 一覧には登録済みの最終需要家が並びます。行をクリックすると詳細画面が開きます。
- 検索ボックスに **社名・コード** を入れて絞り込めます。状態（有効 / 無効）でも絞り込めます。

## 新規作成

一覧右上の「新規作成」から登録します。入力項目は顧客とほぼ同じです。

**基本情報**

- **名称**（日本語は必須、英語は任意）/ **国** / **フリガナ** / **略称** / **法人番号** / **有効**。
- **AI照合名** … 注文書の自動読み取りで社名を照合するための別名リスト。

**住所・連絡先**

- **郵便番号** / **住所** / **電話番号** / **FAX** / **メールアドレス** / **Webサイト** / **備考**。

**需要家情報**

- **業種** … 例：自動車部品。

**BPコード**（`BP-NNNNN`）は保存時に自動で採番されます。

## 詳細画面

- 基本情報・需要家情報・担当者などが表示されます。
- 右上のメニューから **編集** / **無効化** / **削除** ができます。

## 用語

- **最終需要家** … 製品を実際に使う会社。注文をくれる顧客とは別のことがあります。
- **業種** … その会社の事業分野。
- **BP（取引先）** … 顧客・仕入先・需要家などをまとめて管理する会社の単位。

初めての方は [スタートマニュアル](/docs/start) もあわせてご覧ください。
